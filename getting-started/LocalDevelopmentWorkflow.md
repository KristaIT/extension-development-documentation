# Local development workflow

## Overview

This page provides a quick start workflow. For the detailed version, see the Development section.

## Recommended workflow

1. Implement or update a single catalog request (one behavior at a time).
2. Run unit tests locally.
3. Build the JAR.
4. Upload/deploy to a test environment.
5. Configure the invoker in the Setup tab.
6. Run a simple smoke test request.
